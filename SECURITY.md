# Security Policy

## Supported Versions

We only provide security updates for the latest stable version of Own Pay. Please ensure you are running the most recent release to stay protected.

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < v0.1  | :x:                |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

If you discover a security vulnerability within Own Pay, please report it privately via email to **security@ownpay.org**. 

### What to include in your report:
- A detailed description of the vulnerability.
- Steps to reproduce the issue (including proof-of-concept code if possible).
- The potential impact of the vulnerability.
- Your name and how you would like to be credited (optional).

### Our Process:
1. We will acknowledge receipt of your report within 48 hours.
2. We will investigate the issue and provide a timeline for a fix.
3. Once the fix is ready, we will coordinate a public disclosure date with you.
4. We will credit you for the discovery (unless you prefer to remain anonymous).

### Guidelines:
- Please give us reasonable time to investigate and fix the issue before making any information public.
- Avoid privacy violations, destruction of data, and interruption or degradation of our service.
- Only interact with accounts you own or with the explicit permission of the account holder.

---
Thank you for helping keep Own Pay and its users safe.
