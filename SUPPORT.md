# OwnPay Support Policy

Thank you for using **OwnPay**! We want to ensure you have the best possible experience running your sovereign payment gateway orchestration engine. Below are the official support channels and resources available for the community and enterprise users.

---

## 💎 Support Channels

Choose the channel that best matches the nature of your query to get the fastest resolution:

### 📖 1. Documentation & Self-Help
Before opening an issue or reaching out for support, please review our official documentation resources:
*   🌐 **Official Website**: [ownpay.org](https://ownpay.org) — Announcements, release notes, and general information.
*   📚 **API Reference**: [docs.ownpay.org](https://docs.ownpay.org) — Detailed endpoint parameters, request/response models, and developer guides.
*   📖 **Learn Platform**: [learn.ownpay.org](https://learn.ownpay.org) — Step-by-step tutorials, custom domain setup, gateway setup, and deployment guides.
*   📝 **Blog**: [blog.ownpay.org](https://blog.ownpay.org) — Visual walkthroughs, fintech integration patterns, and platform updates.

### 💬 2. General Queries & Discussions
For questions regarding configuration, general installation help, architecture designs, or feature feedback:
*   💬 **GitHub Discussions**: Join our GitHub Discussions forum to chat with other developers and share patterns.
*   👥 **Facebook Community Group**: Connect with the community via the [OwnPay Facebook Group](https://www.facebook.com/groups/ownpay.org).

### 🐛 3. Bug Reports & Feature Requests
If you have identified a technical bug, database regression, or would like to request a new core feature:
*   📂 **GitHub Issues**: Open an issue using our structured bug or feature templates in the [OwnPay Repository](https://github.com/own-pay/OwnPay/issues).
*   *Please provide detailed logs, environment configurations, and steps to reproduce.*

### 🛡️ 4. Security Vulnerabilities
If you suspect or discover a security issue or credential leak:
*   🔒 **Private Security Disclosure**: Report security vulnerabilities privately. **Do not post security issues publicly.** Refer to our [Security Policy](SECURITY.md) and email [ping@ownpay.org](mailto:ping@ownpay.org).

### 📮 5. Enterprise Support & Sponsoring
For companies seeking dedicated SLAs, private security audits, custom gateway integrations, or sponsoring:
*   📧 **Direct Support Email**: Contact us directly at [ping@ownpay.org](mailto:ping@ownpay.org).
